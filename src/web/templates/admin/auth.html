Step 1 · Create an OAuth client in Google Cloud Console
  1. Open console.cloud.google.com / Credentials (sign in with a betterplace.co.in account).
  2. Click + Create Credentials → OAuth client ID.
  3. If prompted, configure the OAuth consent screen first: User type = Internal, app name = BetterPlace Co-Pilot, user support email = yours.
  4. Back at "Create OAuth client ID": Application type = Web application, name = Co-Pilot Web.
  5. Add these Authorized redirect URIs (both — one for localhost dev, one for the public tunnel URL you'll set up next):
    {{ state.localhost_callback_url }}
    {{ state.callback_url }}
  6. Click Create. Copy the Client ID and Client Secret it shows you.
Step 2 · Paste credentials here — we'll write them to .env
Target file: {{ state.env_path }} · permissions reset to 600.
Step 3 · Restart uvicorn so .env reloads
The app reads .env only at boot. After saving above, stop uvicorn (Ctrl-C in the terminal where you started it) and run it again:
$ make run
# or: uvicorn src.web.app:app --reload
Once back up, you'll see the "✓ enabled" tag at the top of this page.
Step 4 · Test the login
In a separate browser (or incognito), open /auth/login. You should get redirected to Google, then back here as the new user. Their entry will appear under /admin/users with the View-only role auto-assigned.
