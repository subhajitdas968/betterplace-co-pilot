Start loop — keeps running in the background every {{ state.interval_sec }}s. Survives uvicorn reloads.
Run one sync now — single pass, exits when done. Doesn't affect the loop. Useful when you want a refresh without leaving anything running.
Where does the data go?
Tickets and customers land in data/copilot.db. The sync is incremental — only tickets updated_at since the last successful run are pulled. Run-once is safe to repeat; it won't duplicate anything. Field and form definitions are also refreshed each pass so new ticket fields in Zendesk show up here within {{ state.interval_sec }}s.
